Проект: Сайт + мобильное приложение для управления коттеджным посёлком «Берёзка»

Заказчик: ТСН (товарищество собственников недвижимости).
Пользователи: жители посёлка и администрация ТСН.

Нужно два варианта оценки: MVP и Full.

---

ОБЩЕЕ ОПИСАНИЕ

Цифровая платформа для эксплуатации коттеджного посёлка: коммуникации между жителями и УК, подача и отслеживание заявок, финансы (начисления, оплата, показания счётчиков), управление доступом (пропуска, охрана), сервисные услуги.

Платформы: веб-сайт (адаптивный) + мобильное приложение (iOS/Android). Интеграции: 1С (бухгалтерия ТСН), Битрикс24 (обработка заявок, открытые линии для чата), Telegram (бот + канал), эквайринг (СберPay, оплата картой, QR).

Роли: Пользователь (житель), Администратор (ТСН).

---

ФУНКЦИОНАЛЬНОСТЬ — FULL-ВЕРСИЯ

1. Коммуникационный модуль
- Лента новостей посёлка с анонсами мероприятий и объявлениями
- Уведомления о плановых работах, отключениях, ЧС
- Электронная документация: правила проживания, регламенты, решения собраний
- Чат с управляющей компанией (через открытые линии Битрикс24)
- Telegram-канал: дублирование новостей и уведомлений
- GPT-шаблонизатор для генерации новостей администратором

2. Система управления заявками
- Подача заявок с фото и описанием проблемы
- Отслеживание статуса в реальном времени
- История обращений с оценкой качества
- Связь с ответственным исполнителем
- Telegram-бот для подачи заявок
- Бэкенд-интеграция с Битрикс24 для обработки заявок

3. Финансовый блок
- Просмотр начислений за услуги УК
- Онлайн-оплата сервисного сбора и допуслуг (СберPay, карта, QR)
- Передача показаний счётчиков электроэнергии
- История платежей и электронные квитанции (печать/скачивание)
- Просмотр финансовой отчётности по расходованию средств (прозрачность)
- Обмен данными с 1С ТСН (начисления, оплаты, сальдо)
- Уведомления: напоминание об оплате, задолженности, необходимости передать показания

4. Управление доступом и безопасностью
- Система пропусков для гостей: заявка на проезд машины (номер авто) → уведомление охране
- Telegram-бот для запроса «впустить автомобиль с номером X» → охрана получает в ТГ
- Просмотр камер видеонаблюдения в общественных зонах (стрим)
- Тревожная кнопка — экстренная связь с охраной

5. Сервисные функции
- Каталог и заказ дополнительных услуг: сантехник, электрик, клининг

6. Функции пользователя (личный кабинет)
- Авторизация, просмотр лицевого счёта (номер ЛС, адрес, ФИО, телефон, email, баланс)
- Редактирование профиля (телефон, email, пароль)
- Внесение показаний счётчика ЭЭ
- История начислений и оплат за выбранный период
- Просмотр и печать квитанции
- Настройка уведомлений (показания, квитанции на email)
- Согласие на обработку ПД
- Раздел «Голосование» — просмотр и участие в голосованиях

7. Функции администратора
- Поиск жителя по ФИО, номеру ЛС
- Фильтры: посёлок, член ТСН, адрес, размер долга
- Отправка уведомлений о задолженности (email, SMS, робозвонок)
- Ответ на обращения с прикреплением документов
- Управление контентом сайта: новости, фото, видео, файлы, опросы, голосования
- Выгрузка/загрузка документов, печать отчётов

8. Публичная часть сайта (без авторизации)
- Разделы: Новости, Контакты, Частые вопросы

---

ФУНКЦИОНАЛЬНОСТЬ — MVP

1. Коммуникационный модуль
- Лента новостей с объявлениями
- Уведомления о работах и отключениях
- Электронная документация
- Чат с УК
- Telegram-канал

2. Заявки
- Упрощённая подача заявок через Telegram-бот (данные фиксируются в системе)

3. Финансовый блок — отсутствует в MVP

4. Безопасность
- Telegram-бот для оповещения охраны (запрос на пропуск авто)

5. Сервисные функции — отсутствуют в MVP
